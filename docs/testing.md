# 批量测评工具说明

对话系统改动一多，手工一条条测跟不上，所以做了这个工具批量跑用例、自动判分。判分为什么分层、为什么不能只靠关键词，见[复盘的测评部分](retrospective.md#怎么验证它好使测评的两版演进)，这里只讲怎么用。

## 启动

```bash
pip install flask pandas openpyxl requests
python3 testing/test_console.py     # 打开 http://localhost:8100
```

前提：Dify 和数据服务（`server/bank_api.py`）都在跑，否则用例发不出去、核验查不到。

## 三页流程

**第 1 页 · 启动检测**
两个状态灯，确认 Dify 和数据服务都连得上。绿灯才往下走。

**第 2 页 · 选测评集 + 配判分模型**
- 勾选一个或多个测评集（`data/testset.xlsx` 是现成的），也可以上传自己的
- 配判分模型：填 OpenAI 兼容端点和 key，点「拉取全部」选模型。留空则退回关键词判分（不准，只能靠人工兜）
- 判分提示词可以改，占位符是 `{输入}` `{标准答案}` `{实际回复}`

**第 3 页 · 跑 + 看结果**
- 设并发数（同时跑几个用例，3-6 之间稳妥），点开始
- 跑完看四个指标：正确率、错误率、session 数、总轮次
- 明细表逐条列：输入 / 标准答案 / 实际回复 / 判定 / 申诉 / 标注
- 能筛「只看失败 / 只看模糊」，能导出 CSV

## 判分怎么定的

三层，优先级从高到低：

1. **禁止词（代码硬判）**：命中就直接失败，不经过大模型。安全类用例用这个——比如防泄露用例里，回复只要出现真实卡号，无条件失败。安全红线不交给另一个模型去「觉得」。
2. **大模型判分**：对比标准答案和实际回复，看意思对不对，不管措辞。解决关键词判分「说法一变就误判」的问题。
3. **人工申诉**：机器判完，你可以点 ✓/✗ 覆盖，最终分按人工裁定算。机器解决「量」，人工解决「准」。

明细里每条会标「判定依据」是大模型还是关键词，一眼看出判分模型有没有生效（没配 key 会静默退回关键词）。

## 测评集怎么写

Excel，一行一轮对话，列如下：

| 列 | 说明 |
|---|---|
| 用例编号 | 多轮对话共用一个编号 |
| 测试类型 | 分类标签，用于分组看正确率 |
| 对话轮次 | 第几轮 |
| 输入内容 | 这轮发给 agent 的话 |
| 标准答案 | 描述这轮该达成什么（给判分模型对照，写目的不写字面） |
| 期望包含 | 关键词判分用，`|` 分隔，回复含任一即通过 |
| 禁止包含 | `|` 分隔，命中即失败（放真实卡号这种红线） |
| 判分轮次 | 1=参与判分，0=只发消息不判（多轮的中间轮填 0，只判最后一轮） |

多轮用例：同一个用例编号下按轮次排好，工具会在同一个会话里连续发，只对「判分轮次=1」的行打分。

## 生成测评集

`scripts/gen_batch_testset.py` 会读虚拟库、挑代表性数据（锁定/冻结/正常客户、各种转账状态）自动生成 26 个用例，覆盖核验、转账、安全（注入/越狱/套取）、情绪、知识边界等场景。改这个脚本就能加自己的用例。
