# 银行智能客服：任务型多轮对话系统

基于 Dify 编排的银行客服 Agent，面向单用户多轮会话的任务型对话场景（身份核验、转账查询、产品咨询），配套独立的数据查询服务和一套批量测评工具。

做这个项目最主要想清楚的一件事，是把"精确的事"和"模糊的事"分开处理：核验身份、查转账这种要精确匹配的，走独立查询接口，客户的原始数据不进大模型；产品咨询这种可以模糊的，才用向量检索。这样卡号之类的敏感信息从根上就不会被大模型说漏。

## 能做什么

- 分意图处理：产品咨询、账户业务、转账查询、转人工、闲聊，多轮之间能接得住上文
- 核验身份时能从自然语言里抽信息——一次说全、分几轮说、乱序说、夹在情绪化的话里说都行，缺哪样问哪样，什么都没给才弹表单
- 转账查询支持口语时间（"昨天"会换算成具体日期）
- 产品咨询只依据知识库回答，库里没有的会说明而不是编
- 遇到愤怒/着急的用户先安抚再办业务
- 对注入、越狱、套取信息这类做了防护
- 批量测评用大模型当裁判判语义，安全类用硬规则，最后可人工复核

## 架构

```
用户输入
 → 取当前日期
 → 意图分类（带记忆，能接上文）
 → 分流
     闲聊/越界   → 闲聊应答
     转人工      → 转人工
     产品咨询    → 向量检索 → 生成回答
     账户业务    → 再分核验 / 转账
        核验 → 参数提取 → 代码合并槽位 → 按齐没齐分三路
                 齐了   → 查核验接口 → 组织回复
                 缺一点 → 追问缺的
                 全没有 → 先共情 → 弹表单
        转账 → 同样一套 → 查转账接口
 → 汇总 → 统一出口
```

精确数据放在独立的 Flask 服务（`server/bank_api.py`）后面，它实时读 `data/` 下的 Excel，改 Excel 就等于更新数据。以后接银行真系统只换这个服务，编排不用动。

## 目录

```
frontend/   聊天前端（原生 HTML/CSS/JS）
server/     数据查询服务
testing/    批量测评（网页控制台 + 命令行）
data/       customers / transfers / product_manual / testset
scripts/    数据和测评集的生成脚本
docs/       复盘、并发设计、开发日志
```

## 跑起来

```bash
# 数据服务
pip install flask pandas openpyxl requests
python3 server/bank_api.py                 # :8000

# Dify：按架构搭好并发布，HTTP 节点填 http://host.docker.internal:8000
#       （Dify 在 Docker 里，不能写 localhost）

# 前端：frontend/app.js 填 DIFY_URL / DIFY_KEY，浏览器打开 index.html

# 批量测评（可选）
python3 testing/test_console.py            # :8100
#   大模型判分要配 DASHSCOPE_API_KEY，环境变量或页面里填都行
```
